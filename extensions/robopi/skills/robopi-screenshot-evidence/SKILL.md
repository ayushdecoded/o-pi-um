# RoboPi Screenshot Evidence Skill

Use this skill for UI-facing RoboPi work or any task where visual proof helps review.

## Evidence workflow

1. Identify the screen/state that proves the change.
2. Use Pi browser tools when available to open, inspect, and capture the page.
3. Save or reference the screenshot path if a file is produced.
4. Record concise evidence in the assigned task.
5. Mention caveats, e.g. mocked data, local-only environment, or unauthenticated state.

Keep evidence short and review-focused.
